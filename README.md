# Discord Music Bot 🎵

Потужний, стабільний та сучасний музичний бот для Discord, що використовує **Slash-команди**, підтримує **Automix**, розумне керування чергою (через інтерактивний UI), а також має власну систему статистики та збереження історії.

## 🌟 Основні можливості

- **Slash-команди (`/`)**: Бот повністю перейшов на сучасні Slash-команди Discord. Більше ніяких префіксів `!`!
- **Інтерактивний плеєр (UI)**: Зручні кнопки під повідомленням плеєра для керування: *Відтворити/Пауза, Пропустити, Повернутись, Показати чергу, Гучність та Історія*.
- **DJ & Automix**:
  - **Automix**: Дозволяє боту автоматично підбирати схожі треки та продовжувати програвання після закінчення черги.
  - **DJ Persona**: Бот може коментувати треки в стилі різних персон (chill, energetic, funny).
  - Гнучкі налаштування міксу (Fade-out, стратегії підбору) через інтерактивне меню.
- **Auto-Resume (Відновлення)**: При перезапуску бота, він автоматично запам'ятовує чергу та відновлює програвання на тих серверах, де музика грала до вимкнення.
- **Статистика та Історія**: Бот записує історію прослуховувань у власну базу даних (SQLite) та дозволяє переглядати статистику і топ-треків.

## 🛠 Технологічний стек
- **Python 3.12+**
- **[discord.py](https://github.com/Rapptz/discord.py)** — основний фреймворк для бота.
- **yt-dlp** — для пошуку та завантаження аудіо (YouTube, Spotify, SoundCloud тощо).
- **FFmpeg** — для обробки аудіо-стріму (fade-in/fade-out).
- **SQLite (aiosqlite)** — асинхронна база даних для історії, черг та налаштувань.
- **Docker / Docker Compose** — для зручного деплою (включно зі збіркою FFmpeg).
- **Pytest** — 100% покриття тестами (Unit та Integration).

## 🚀 Запуск бота

### Спосіб 1: Локальний запуск (Без Docker)
1. **Клонуйте репозиторій:**
   ```bash
   git clone <URL_РЕПОЗИТОРІЮ>
   cd discord-music-bot
   ```

2. **Встановіть FFmpeg:**
   Завантажте та встановіть [FFmpeg](https://ffmpeg.org/download.html), додавши його до `PATH`. 
   На Linux: `sudo apt install ffmpeg`.

3. **Створіть віртуальне середовище та встановіть залежності:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # Mac/Linux
   # venv\Scripts\activate   # Windows
   pip install -r requirements.txt
   ```

4. **Налаштуйте `.env`:**
   Створіть файл `.env` у кореневій папці та додайте свій токен:
   ```env
   DISCORD_TOKEN=ВАШ_БОТ_ТОКЕН_ТУТ
   ```

5. **Запустіть бота:**
   ```bash
   python main.py
   ```

### Спосіб 2: Запуск через Docker
У проекті налаштовано `Dockerfile` та `docker-compose.yml`, які автоматично підтягують FFmpeg та всі залежності.
1. Вкажіть токен у файлі `.env`.
2. Запустіть контейнер:
   ```bash
   docker-compose up -d --build
   ```

## 🎮 Команди (Slash Commands)
Просто введіть `/` у чаті Discord, щоб побачити список команд:
- `/play <назва або URL>` — Додати трек/плейлист у чергу та почати відтворення.
- `/queue` — Показати поточну чергу (з пагінацією та можливістю видаляти/переміщувати треки через інтерактивне меню).
- `/history` — Показати історію програних треків.
- `/stats` — Ваша статистика прослуховувань за певний період.
- `/automix` — Увімкнути/вимкнути авто-підбір наступних треків.
- `/mix` — Відкрити детальне меню налаштувань DJ та Automix.
- `/volume` — Змінити гучність від 0 до 200%.

## 📂 Структура проекту
*   `main.py`: Точка входу, ініціалізація бота, завантаження модулів.
*   `discord_music_bot/`: Основний пакет логіки.
    *   `cogs/slash_music_cog.py`: Реєстрація Slash-команд та зв'язка подій.
    *   `services/`: Бізнес-логіка (плеєр, черга, automix, історія, відновлення черги).
    *   `views/`: UI-компоненти (кнопки плеєра, інтерактивні модалки, селекти).
    *   `database.py`: Робота з SQLite (репозиторій).
*   `tests/`: Комплексний набір тестів (`pytest`), що покриває 100% функціоналу.
*   `data/`: Папка для бази даних `music_bot.db` (генерується автоматично).
